## 📌 프로젝트 계획서

### 1. 프로젝트 개요

* **프로젝트명**: IFITV
* **목표**: AI 기반 하이브리드 추천 엔진을 활용하여 TV·VOD 콘텐츠를 사용자 맞춤형으로 추천하고, 멀티 프로필 및 손동작 제스처 기반의 차별화된 시청 경험을 제공
* **기간**: 2025년 5월 \~ 2025년 8월

### 2. 프로젝트 일정

* **기획 및 설계**: 5월 중순
* **개발 환경 구축 및 데이터 수집/전처리**: 5월 말 \~ 6월 중순
* **추천 시스템 개발**: 6월 중순 \~ 7월 초
* **UI/UX 구현 및 테스트/배포**: 7월 초 \~ 7월 말
* **최종 발표 및 보고서 작성**: 7월 말 \~ 8월 초

## 📦 모델 정의서

### 1. 데이터 모델

* **User**: 이메일, 비밀번호, 성별/연령대 등
* **Profile**: 사용자 프로필 정보 최대 4개
* **VOD/Live Metadata**: 제목, 장르, 썸네일, 줄거리 등
* **Watch History / MyList**: 사용자 콘텐츠 선호 로그
* **임베딩**: TF-IDF, KoBERT, ResNet50 기반 콘텐츠/썸네일 임베딩

### 2. 추천 모델

#### \[1] 콘텐츠 기반 하이브리드 추천 모델

* **구성**: TF-IDF, KoBERT, 장르/서브장르, 사용자 행동 데이터
* **모델**: Logistic Regression
* **평가**: HR\@10, NDCG\@10, MRR\@10

#### \[2] 이미지 유사도 딥러닝 하이브리드 모델

* **구성**: ResNet50(썸네일) + KoBERT(줄거리) + 장르/서브장르
* **가중치**: 썸네일(0.3), 줄거리(0.3), 장르(0.2), 서브장르(0.2)
* **기능**: 유사 콘텐츠 Top-N 추천

## ⚙️ 성능 평가 결과서

### 1. 테스트 환경

* **서버**: AWS EC2
* **DB**: RDS (MySQL), Redis
* **네트워크**: 1Gbps

### 2. 정량적 성능

| 지표       | TF-IDF 모델 | KoBERT 하이브리드 모델 | 향상 배수   |
| -------- | --------- | --------------- | ------- |
| HR\@10   | 0.07      | 0.372           | 약 5.3배  |
| NDCG\@10 | 0.016     | 0.253           | 약 15.8배 |
| MRR\@10  | 0.012     | 0.227           | 약 18.9배 |

### 3. 주요 시사점

* KoBERT 임베딩 도입 → 문맥/감정 기반 추천 정확도 상승
* 행동 기반 피처 추가 → 개인화 정밀도 대폭 향상
* 단일 TF-IDF 한계 확인 → 의미 정보(BERT) 기반 하이브리드 구조 필수
